# Fluxo Agent API

## Overview

Use this skill to call the Fluxo Agent API from any repo that has access to a tenant key. Prefer it when MCP is unavailable, when another agent needs explicit `curl` commands, or when you want a deterministic helper to build and execute authenticated HTTP requests.

## Workflow

1. Confirm the current project has `.env.local` with `FLUXO_AGENT_KEY`.
2. Optionally read `references/http-patterns.md` for endpoint patterns and payload examples.
3. Use `scripts/fluxo_api.py curl ...` when you want a shell command to show, copy, or adapt.
4. Use `scripts/fluxo_api.py request ...` when you want to execute the call and inspect JSON output directly.
5. For writes, always send `_metadata.changeReason` in the JSON body.

## Environment Contract

- Require `FLUXO_AGENT_KEY` in the consumer repo's `.env.local`. Do not hardcode keys in prompts, scripts, or committed files.
- Accept `FLUXO_AGENT_API_URL` as optional. Default to `http://localhost:3005/api/agent` when absent.
- Accept `FLUXO_AGENT_NAME` as optional. Default to `Codex Skill`.
- Search upward from the current working directory for `.env.local`, then `.env`. If `FLUXO_AGENT_KEY` is still missing, stop and tell the user to add it.

## Preferred Commands

Generate a curl command:

```bash
python scripts/fluxo_api.py curl GET /tasks --query status=DOING --query limit=10
```

Execute a request immediately:

```bash
python scripts/fluxo_api.py request GET /projects --pretty
```

Create a mutation payload from a file:

```bash
python scripts/fluxo_api.py request POST /tasks \
  --data-file payload.json \
  --pretty
```

## Rules

- Prefer the bundled Python helper over handwritten curl when there are query params, JSON payloads, or repeated calls.
- Keep paths relative to `/api/agent`, for example `/tasks`, `/projects`, `/docs/<id>`.
- Preserve the `Authorization: Bearer ...` and `X-Agent-Name` headers generated by the helper.
- Read the live API schema with `GET /api/agent` when you need the exact request/response contract.
- When a command fails with `AUTH_ERROR`, assume `FLUXO_AGENT_KEY` is wrong, missing, or rotated.
- Do not store real tenant keys in the repo. Use `.env.local` only.

## Resources

- `scripts/fluxo_api.py`: resolve envs, generate curl commands, or execute requests.
- `references/http-patterns.md`: endpoint patterns, env contract, and JSON payload examples.
